3. Why Layered Architecture?

Layered architecture divides the LMS into three main parts:

  • Presentation Layer: Handles user interaction (HTML/JS UI).
  • Business Logic Layer (BLL): Contains rules for adding students, courses, and managing assignments/grades.
  • Data Access Layer (DAL): Manages data storage (in-memory here, can later connect to DB).

This design makes the LMS more scalable, maintainable, and secure. Changes in one layer don’t affect others, making it ideal for university systems that evolve over time.
